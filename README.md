# 3D-Basketball-Court-Scene-C-and-OpenGL
3D Basketball Court Scene built in C++ and OpenGL. Features modular object classes, custom shaders, lighting, textures, and full camera controls. Includes court, hoop, ball, and net modeled from geometric primitives. Demonstrates 3D rendering, transformations, and scene design. With the Design Decisions documents. 

# How I approach designing software
I approach software design by breaking complex problems into modular, manageable components. For this project, I treated each object in the scene—the court, ball, hoop, backboard, rim, and net—as its own class with clear responsibilities. This helped me think in terms of reusable structures, transformations, and rendering logic. I also relied on reference images and sketches to guide proportions and spatial relationships before writing any code.

# New design skills I developed
Working on this project strengthened my ability to translate 2D references into accurate 3D models. I learned how to design objects using geometric primitives, how to plan transformations in world space, and how to structure a scene so that each object can be independently positioned, rotated, and scaled. I also improved at designing lighting and material properties to achieve realistic visual results.

# The design process I followed
My design process followed an iterative, milestone‑based workflow. I began by identifying the core objects needed for the scene, then created each one as a separate class. I used incremental testing to verify that each object rendered correctly before integrating it into the full scene. Once the base geometry was complete, I refined textures, lighting, and camera controls to enhance realism and usability.

# How these design tactics apply to future work
This approach—modular design, incremental building, and reference‑driven modeling—applies directly to future software engineering, game development, and simulation projects. Breaking work into components reduces complexity, makes debugging easier, and supports scalability. The habit of grounding design decisions in real‑world references will also help me create more accurate and believable systems.

# How I approach developing programs
I approach development by building core functionality first, then layering complexity through iteration. For this project, I started with rendering a simple plane, then added objects one at a time. I used consistent testing, debugging, and refactoring to keep the code clean and maintainable. I also relied heavily on transformation matrices, shader logic, and camera movement to bring the scene to life.

# New development strategies I used
I learned how to manage a full 3D rendering pipeline using OpenGL, including VAOs, VBOs, shaders, and texture mapping. I also improved at writing reusable transformation functions, handling lighting calculations, and structuring code so that each object had its own initialization and draw routines. Implementing camera controls taught me how to integrate user input with real‑time rendering.

# How iteration factored into development
Iteration was essential. Each milestone required me to refine earlier work—adjusting object proportions, improving lighting, fixing camera clipping, and reorganizing code for clarity. The project evolved from simple shapes into a polished scene through repeated testing and refinement.

# How my development approach evolved
Early milestones focused on getting objects to render correctly. As the project progressed, I shifted toward improving realism, optimizing structure, and ensuring the scene met rubric requirements. By the end, my workflow was more systematic, with better planning, cleaner code, and more deliberate use of transformations and materials.
